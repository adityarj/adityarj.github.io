How we did the project

We wanted to work on a visualisation using Instagram. However, we did not know our question yet. As Instagram’s official API (application programming interface) is restrictive, we have to ensure that our question will be within the scope of data we can collect. This year, iLight was an exhibition held from 3rd to 26th March at Marina Bayfront. As it was a popular event with Instagram users and its exhibitions were all instagram worthy, we felt that iLight is an event that can be visualised through Instagram. As such, our question became:

“Can we see trends for iLight 2017 through Instagram?”

Getting the Data

Our data came from public accounts in Instagram. As we had restrictions (explained below), we used a combination of node.js and Instagram API to scrap photos tagged #iLight. Next, comments and date of post was scrapped using javascript and downloaded as a CSV file.

Cleaning and Classifying Data

The original CSV file was badly formatted and such was cleaned using R before being reexported. The dates were in time since epoch and thus had to be converted to date format.
Thee photographs were manually sorted as shown below:

- Photo Num: corresponds to the names of the photos that we had queried
- Time: general time of the day that the photo was taken
(day/night/unknown)
- Group type: the type of human subject found in the photo
(single female/single male/couple/group)
-exhibition: name of the exhibit/location of photo. If the location cannot be identified, "unknown" was used. If the location was in the Marina Area but not of iLight, a "marina" tag was used.

Problems and Limitations Faced

The official Instagram API is quite restrictive due to us not being an approved developer.

Restrictions that affected us are

  • Any developers (sandbox mode) must be approve by Instagram before release of their apps (live mode)

  • Hence, we had limited access to pictures and number of users

To get around we found a nodejs wrapper on github that emulates a virtual app and queries Instagram’s internal API. As such, we were able to bypass the limits. That's how we ended up with more than 1000 photos for our data sample.


Another limitation we faced is that iLight is an exhibition. iLight does not have a permanent geo-tagged location (i.e, coordinates) on Instagram. Hence

  • We had to decide on a common location tag to query: #ilight

  • We cannot rely on coordinates to know which photos were taken at which exhibits

Thus, this resulted in all photos having to be manually sorted, using iLight's official map to guide us on how each exhibits looked like.
